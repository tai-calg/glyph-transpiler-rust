# 設計書

## 1. 目的

利用者が繰り返し記述する概念を、短く一意な記号へ写像する。

```text
頻出概念 ──固定写像──> 記号 ──決定的変換──> Rust構文
```

このツールの価値は、Rustの全機能を置き換えることではない。設計初期に頻出する次の概念を圧縮することにある。

1. 値を同時に保持する型
2. 排他的な状態
3. 型から型への変換
4. 外部作用との境界
5. 同じ式または単語の反復

## 2. 採用した記号

```text
@  式単語マクロ
*  積型
+  直和型
>  変換
!  外部作用
```

理由:

- `@`は通常の宣言と異なるコンパイル前の定義を示す
- `*`と`+`は、それぞれ積型と直和型の意味に対応する
- `>`は入力から出力への向きを表す
- `!`は通常の純粋計算とは異なる境界を明示する

## 3. コンパイラ構成

```text
.glyph source
    │
    ├── macro collector
    │       └── @NAME=expression
    │
    ├── expression lexer
    │
    ├── token macro expander
    │       ├── exact identifier matching
    │       ├── recursive resolution
    │       ├── cycle detection
    │       └── parenthesized insertion
    │
    ▼
line parser + Pratt expression parser
    │
    ▼
declaration / expression AST
    │
    ▼
validation
    │
    ▼
Rust generator
```

マクロは文字列置換ではない。式lexerが生成した`IDENT`トークンだけを置換する。このため、`@R=1`が`Receipt`内部の文字`R`を書き換えることはない。

## 4. マクロの意味

### 4.1 対象

マクロは式中の識別子を式トークン列へ置換する。

```text
@MAX=1000
@CAP=min(x,MAX)
```

型、宣言名、フィールド宣言は置換しない。DSLの型構造は通常構文のまま保持する。

### 4.2 優先順位保存

置換トークン列を括弧で囲む。

```text
@A=x+1
A*2
```

は`(x+1)*2`として解析する。Cの単純な関数形式マクロで起こる演算子優先順位の変化を許さない。

### 4.3 決定性と停止性

- マクロ定義は全件収集してから展開する
- 宣言順は意味へ影響しない
- 同名定義を禁止する
- 循環参照を禁止する
- 最大展開深さを64とする
- 最大展開トークン数を4096とする

## 5. 静的検査

DSL側で検査する項目:

- マクロ名と本体の妥当性
- マクロ重複、循環、過大展開
- マクロ名と宣言・variant名の衝突
- トップレベル名の重複
- enum variant名の全プログラム内衝突
- structフィールド名の重複
- variantフィールド名の重複
- structおよびvariantコンストラクタの引数数
- ガード関数の最後に`_`が一つだけあること
- 未閉鎖括弧
- 不正なトークン

Rust側へ委譲する項目:

- 型整合性
- 所有権と借用
- trait制約
- 外部関数シグネチャの実装一致
- 対象CPUとABI
- マクロ本体に現れる通常識別子の名前解決

## 6. 外部作用境界

DSLで外部作用を実装しない。`!name(...)`は境界の宣言であり、呼出しを`crate::host::name(...)`へ固定する。

目的:

- 純粋ロジックの短い記述を維持する
- ハードウェア依存コードをRust側へ隔離する
- DSLを所有権、unsafe、非同期ランタイムの複雑さから切り離す

## 7. 決定性

同じ入力ソースと同じコンパイラ版からは、同一のRust文字列を生成する。時刻、乱数、環境変数を生成結果へ含めない。

## 8. 非目標

- Rust全体の短縮構文を作ること
- Cプリプロセッサ互換を実装すること
- 識別子の一部分を置換すること
- 型またはトップレベル宣言をマクロ展開すること
- 引数付きマクロを実装すること
- Rustの型検査器を再実装すること
- 生成コードを人間が直接編集すること

## 9. 拡張順序

必要が実例で3回以上反復した場合のみ追加する。

優先候補:

1. 引数付き式マクロ
2. 定数専用宣言
3. `match`による値パターン
4. 参照引数の明示
5. ジェネリック関数
6. プロパティ宣言からRustテスト生成

新しい構文を先に追加しない。頻出概念が確認された後に、その概念へ一つの記号を割り当てる。
